## Preview page checklist

Use this checklist when building a preview shell or picker for the five generated directions.

- [ ] All five variations are rendered from real source files, not temporary DOM injection
- [ ] Each variation is labeled consistently with its comparison-table row
- [ ] The picker can switch among all five options without ambiguity
- [ ] Preview content uses realistic placeholder data or copy
- [ ] Responsive behavior is easy to inspect
- [ ] Primary and secondary actions are clickable or visually complete
- [ ] Empty / loading / validation states are represented when the artifact needs them
- [ ] The preview surface makes it obvious which option is currently active
- [ ] File names map cleanly to variation labels
- [ ] The user can choose an option without cross-referencing hidden implementation details
