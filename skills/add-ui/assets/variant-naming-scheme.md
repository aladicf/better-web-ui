## Recommended naming scheme for generated variants

Use stable, deterministic names so previews, file paths, and user choices stay aligned.

## Display naming

- `Variation 1 — [concept name]`
- `Variation 2 — [concept name]`
- `Variation 3 — [concept name]`
- `Variation 4 — [concept name]`
- `Variation 5 — [concept name]`

Concept names should be short, idea-led, and visually descriptive without being gimmicky.

Good examples:

- `Variation 1 — Editorial Split`
- `Variation 2 — Trust-Led Conversion`
- `Variation 3 — Minimal Product Reveal`
- `Variation 4 — Dense Operator Console`
- `Variation 5 — Warm Guided Onboarding`

## File naming

When generating multiple files, prefer deterministic suffixes:

- `HeroVariation1.tsx`
- `HeroVariation2.tsx`
- `HeroVariation3.tsx`
- `HeroVariation4.tsx`
- `HeroVariation5.tsx`

Or for folders:

- `hero-variation-1-editorial-split/`
- `hero-variation-2-trust-led-conversion/`
- `hero-variation-3-minimal-product-reveal/`
- `hero-variation-4-dense-operator-console/`
- `hero-variation-5-warm-guided-onboarding/`

## Selection mapping rule

The visible label, comparison-table row, and source artifact name should all preserve the same variation number.
