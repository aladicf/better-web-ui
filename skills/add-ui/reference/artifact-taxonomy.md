## Artifact taxonomy

Use this reference when classifying the request before generating any UI.

### Section

Examples: hero, features, testimonials, FAQ, CTA, logo cloud, gallery slice.

Focus on:

- how the section supports the page's primary goal
- which content blocks must appear above the fold vs below
- whether the section should lead, support, reassure, or convert

Common states to consider:

- responsive stacking
- missing images or logos
- reduced-content fallback

### Page

Examples: about, contact, login, register, checkout, error page.

Focus on:

- the page's single dominant task
- page-level hierarchy and navigation context
- how the page should open, reassure, and close

Common states to consider:

- loading or skeleton states when relevant
- validation and submission feedback
- empty, error, and success outcomes

### Flow

Examples: forgot password, reset password, verify email, onboarding, cart to checkout.

Focus on:

- the user's next step at every stage
- what information must remain visible across steps
- how to reduce ambiguity, friction, and abandonment

Common states to consider:

- in-progress step state
- incomplete or invalid input
- expired, blocked, or already-complete branches

### Shell

Examples: dashboard shell, settings shell, app navigation framework.

Focus on:

- what the shell should make easier, not just what it should contain
- navigation priority and escape hatches
- content density and layout rhythm

Common states to consider:

- collapsed navigation
- mobile navigation adaptation
- empty or first-run content areas

### Data or content component

Examples: charts, widgets, tables, compare views, timelines, product grids.

Focus on:

- scanability
- sorting, filtering, or comparison needs
- legibility at realistic data density

Common states to consider:

- loading
- empty / no results
- overflow
- long labels, large numbers, and truncation

## Classification checklist

Before generating variants, identify:

1. the artifact type
2. the primary user goal
3. the core content blocks
4. the critical states
5. the placement context inside the broader product
