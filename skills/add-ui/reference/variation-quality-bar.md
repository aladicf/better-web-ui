## Variation quality bar

Use this reference to ensure the five directions are genuinely different and equally reviewable.

## What counts as a distinct variation

A new variation must change more than surface styling.

At minimum, each variation should differ in at least **two** of these dimensions:

- layout structure
- hierarchy strategy
- density
- content framing
- interaction model
- visual tone / personality
- media treatment
- action emphasis

Changing only palette, radius, or icon style does **not** create a new variation.

## Useful axes of variation

- editorial vs product-led
- minimal vs expressive
- conversion-first vs exploration-first
- dense vs spacious
- screenshot-led vs typography-led
- structured enterprise vs warm consumer
- quiet trust-building vs bold statement-making

## Parity checklist

Every option should be comparable on equal footing.

Each generated variation should include:

- the same artifact scope
- responsive behavior
- accessibility expectations
- key interaction states
- realistic placeholders or example copy
- implementation notes when needed

Do not let one option be a sketch while another is polished. Compare like with like.

## Naming guidance

Each variation should have:

- a short memorable name
- a one-sentence thesis
- a clear best-fit use case

Good names communicate concept, not decoration.

Examples:

- Editorial Split
- Trust-Led Conversion
- Minimal Product Reveal
- Dense Operator Console
- Warm Guided Onboarding
